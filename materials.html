Course Materials

Here are some samples of asynchronous content, classroom activities, and independent design assignments from my computer engineering project. As discussed in my teaching philosophy statement, my approach is centered around building lab skills, and each artifact demonstrates the role of that class of object in the course structure: background material in the asynchronous module, scaffolding and practice during synchronous class time, and application of skills to specific engineering problems in independent design assignments.

Artifacts are organized in the order in which they occur in the syllabus.

Asynchronous Module Video: Transistor-Level Logic Gates

This video from the project's first asynchronous module is exemplary of my approach to lecture:

  • I begin with a callback to previous topics.
  • Using careful explanations and visual aids, and restating important points, I describe a concept or model a practice: in this case, showing a technique for anlayzing transistor-level logic gates.
  • Finally, I include a segue into the next part of the module, in which students are given a quiz to practice the analysis technique I just demonstrated (which also gives me evidence of the lecture's efficacy).

Asynchronous Visual Aid: Memory Metaphors

I initially used this sketch in an asynchronous module, and revisited it in the introduction of the corresponding synchronous lesson. The goal is to get students more familiar with abstraction, an important learning objective of this project: while both register file and memory have complicated internal details, we abstract those details away to arrive at a metaphor that models the purpose of each component. This abstract model helps to make sense of instructions like load and store, which move data between the two components, and also frames thinking about the desired functionality of the register file. Thus, I refer back to this table metaphor in the introductory paragraph of the register file design assignment (below).

Independent Design Assignment: Register File

This is the students' second independent design assignment. With these challenges, the students are building skills producing an engineering solution based on provided specifications, within the structure of a (virtual) lab in which their tools and techniques are constrained. Having already begun thinking about the design during classroom scaffolding exercises, students generally submitted very solid drafts of the register file, and I could then provide brief feedback on areas of improvement, to be addressed before the final project submission. The register file assignment also requires the students to build a testbench for their register file circuit, which uses the concept of design hierarchy, a key learning goal for the project.

Classroom Breakout Activity: Assembly Language Programming

I designed this activity to provide scaffolding for the task of writing loops in assembly language. Students worked through this exercise in groups of up to 3 and then we discussed as a class; the structure could then be used as a model for the students to implement their own loops in the subsequent independent design assignment. Thus, I was able to observe the efficacy of this part of the lesson when evaluating students' independent work. The activity also builds familiarity with levels of abstraction in programming, one of the key learning objectives of the project as a whole.

Project-Specific Instruction Set Architecture

I've also included an example of a syllabus-level choice I made: to create a custom instruction set architecture that is accesible to first-year students and only includes those features relevant to the learning goals of this project.

Although I considered using RISC-V, I ultimately decided to make my own ISA tailored to the specific needs of the project. Thus, students could learn to identify the role of opcodes, immediates, and register addresses, without getting confused about details of a real ISA that may or may not be relevant to this project. The simplified opcodes also make for more straightforward control logic, again allowing the students to build the relevant skills without spending time to learn a real ISA (which is not a learning objective of this first-year project).

This document provides both the ISA specification, and notes & examples to help the students learn how to interpret the specification.
